# Циклы, списки, `range`

## Вводная:

Как мы уже установили, список это не то же самое, что `range`; например, `[ 0, 1, 2, 3, 4 ]` не есть то же, что и `range(5)`, однако если перебрать каждый из них с помощью цикла `for`, то они как-будто содержат одни и те же элементы:

```py
a = [ 0, 1, 2, 3, 4 ]
b = range(5)

for i in a:

    print(i)

for i in b:

    print(i)

# Вывод получается одинаковый
```

Но

```py
print(a)
# >>> [ 0, 1, 2, 3, 4 ]
print(b)
# >>> range(0, 5)
```

На занятии 6-го октября мы разбирали следующее задание: при каком-то определенном числе `N`, написать такую программу, которая сделала бы список, содержащий элементы, которые мы увидели бы в `range(N)`, если бы перебрали его через цикл `for`:

```py
N = 5
s = []

for i in range(N):

    s.append(i)

print(s)
# >>> [ 0, 1, 2, 3, 4 ]
```

Теперь, мы могли бы перебрать список `s`, и перебор совпадал бы с перебором `range(N)` — все как мы хотели.

```py
for i in s:

    print(i)

for i in range(N):

    print(i)

# Вывод получается одинаковый
```

При этом, программа зависела от переменной `N` — мы могли бы изменить его на 50 или 100 и получить список с числами от 0 до 49 или от 0 до 99.

> По секрету, это можно сделать быстрее, без цикла `for` и с любым возможным `range` — нужно его обернуть в `list(` и `)`. Например, чтобы сделать список из `range(5)`, нужно написать `list(range(5))`.

## Задание №1

Переписать имеющуюся программу без использования цикла `for`, но с использованием цикла `while`. При этом, результат выполнения программы должен остаться тем же самым — на выходе у нас должна быть **переменная `s`, содержащая список**. То есть:

```py
N = 5
s = []

# <-- Какой-то цикл while здесь

print(s)
# >>> [ 0, 1, 2, 3, 4 ]
```

Разумеется, программа должна зависеть от числа `N`, как и в оригинале.

> Примечание: во время выполнения этого задания можно создавать сколько угодно переменных. Можно делать вообще все, что угодно. Кроме использования цикла `for`, конечно. И `list()`, тоже.

## Задание №2

Переписать имеющуюся программу из вводной части или первого задания, но так, чтобы вместо списка в переменной `s` получалась строка, содержащая то же самое, что выводилось бы в `print` в вводной и первом задании.

До этого, при переменной `N`, равной, например, `5`, в переменной `s` получался список с элементами `0`, `1`, `2`, `3` и `4` — то есть, с числами от `0` до `N - 1`. Выводя такой список с помощью `print` мы бы получали в консоли текст `[ 0, 1, 2, 3, 4 ]`. Программу нужно переписать так, чтобы вместо списка в переменной `s` находился именно этот текст. То есть:

```py
N = 5
s = "[ "

# <-- Какой-нибудь код здесь

s += " ]"

print(s)
# >>> [ 0, 1, 2, 3, 4 ]
```

Как и предыдущая программа, эта должна зависеть от числа `N`.
